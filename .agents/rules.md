# Project Rules

## Planning Mode Required

**ALWAYS present a plan and wait for explicit user approval before:**
- Writing or modifying any file
- Running any terminal command
- Injecting scripts into the browser
- Making any change to the codebase or environment

Acceptable responses during planning: asking questions, describing what you propose to do.

Only proceed to execution when the user explicitly says something like "go", "yes", "do it", "proceed", or approves the plan.
